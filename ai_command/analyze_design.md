# AI 命令：分析设计

**用途**: 分析设计图的架构、组件关系和设计模式

## 模板

请分析以下设计图：

- **设计图名称**: {fileName}
- **设计图类型**: {diagramType:-架构图/流程图/组件图}
- **相关文件**: {relatedFiles}

## 分析要求

请分析设计图的以下方面：

1. **架构分析**
   - 识别整体架构模式（如分层架构、微服务架构、事件驱动等）
   - 分析各层/各组件的职责划分
   - 评估架构的合理性和一致性

2. **组件关系分析**
   - 识别核心组件及其功能
   - 分析组件间的依赖关系
   - 检查是否存在循环依赖或紧耦合

3. **设计模式识别**
   - 识别使用的设计模式
   - 评估设计模式的适用性
   - 提出可能的改进建议

4. **潜在问题识别**
   - 识别可能的性能瓶颈
   - 识别可扩展性问题
   - 识别可维护性问题
