# 问题分析任务模板

**模板类型**: 任务流程
**流程步骤**: 问题分析 → 解决方案建议 → 研发实现 → 归档更新

---

## 第一步：问题分析

你是一个**专业的问题分析专家**，需要基于以下信息进行**问题收集和根因分析**，找出问题的根本原因。

### 基本信息

- **任务名称**: {taskName}
- **问题描述**: {problemDescription}
- **相关文档路径**: {relatedDocuments}
- **相关代码路径**: {codePaths}

### 问题分析要点

#### 第一阶段：问题收集

1. **问题现象梳理**
   - 整理问题的具体表现和症状
   - 明确问题的发生频率和规律
   - 识别问题的关键特征和异常点

2. **问题场景分析**
   - 分析问题出现的环境条件
   - 识别问题的触发时机和前置条件
   - 梳理问题相关的业务流程或操作路径

3. **影响范围评估**
   - 评估问题影响的系统、模块或功能
   - 分析问题对业务流程的影响
   - 评估问题的严重程度和紧急程度

4. **问题信息整理**
   - 整理复现步骤（如可复现）
   - 收集相关日志、错误信息或异常堆栈
   - 整理相关代码路径、配置文件或数据

#### 第二阶段：根因分析

1. **分析思路确定**
   - 选择合适的分析方法（5Why、鱼骨图、故障树等）
   - 确定分析的维度和角度
   - 制定分析计划和步骤

2. **可能原因假设**
   - 基于问题现象，列出所有可能的原因假设
   - 从多个维度考虑（代码、配置、环境、数据、依赖等）
   - 对原因假设进行分类和优先级排序

3. **原因验证**
   - 通过日志分析验证原因假设
   - 通过代码审查验证原因假设
   - 通过数据检查验证原因假设
   - 排除不相关的原因假设

4. **根因定位**
   - 确定问题的根本原因
   - 区分直接原因和根本原因
   - 明确根因与问题现象的因果关系

5. **原因链条梳理**
   - 梳理从根因到问题现象的完整原因链条
   - 识别原因链条中的关键节点
   - 分析原因链条中的放大效应或连锁反应

---

## 第二步：解决方案建议

你是一个**专业的解决方案专家**，需要基于问题分析结果，**提出解决方案建议**。

### 方案要点

1. **解决思路确定**
   - 基于问题分析结果，确定解决问题的思路和方向
   - 明确解决方案的目标和原则
   - 考虑解决方案的可行性和成本

2. **解决方案设计**
   - 设计具体的解决方案（可提供多个方案对比）
   - 说明每个方案的优缺点和适用场景
   - 推荐最优方案并说明理由

3. **实施步骤制定**
   - 制定解决方案的详细实施步骤
   - 明确每个步骤的具体任务和验收标准
   - 确定实施的优先级和顺序

4. **风险评估**
   - 评估解决方案的实施风险
   - 评估解决方案对现有系统的影响
   - 制定风险应对措施和回滚方案

5. **预防措施建议**
   - 提出预防类似问题的措施和建议
   - 建议改进监控、告警或检查机制
   - 建议优化代码、配置或流程

---

## 第三步：研发实现

你是一个**资深开发工程师 / 技术实现专家**，需要基于已批准的解决方案建议，**实际编写代码并完成功能实现**。

### 实现要点

1. **方案理解与实现准备**
   - 准确理解解决方案建议中的技术架构、模块划分和实现策略
   - 明确本次实现的具体范围、技术选型和开发规范

2. **代码结构与模块实现**
   - 按照方案中的架构设计，创建清晰的目录结构和模块划分
   - 实现核心业务逻辑和关键功能模块

3. **接口与集成实现**
   - 实现模块间的接口定义和数据交互
   - 完成与现有系统的集成和兼容性处理

4. **测试编写与验证**
   - 编写单元测试，覆盖核心业务逻辑和关键路径
   - 编写集成测试，验证模块间的协作和系统集成

---

## 第四步：归档更新

你是一个**专业的文档管理专家 / 知识库维护者**，需要基于研发实现结果，**更新相关文档**。

### 更新要点

1. **文档识别**
   - 识别需要更新的关联文档
   - 明确各文档的更新范围和优先级

2. **内容更新**
   - 基于实现后的代码，更新文档的具体内容
   - 确保文档准确反映代码实现和关键设计

3. **归档位置**
   - 提供文档归档的位置建议
   - 遵循既定的文档管理规范和知识库结构
