# 文档设计任务模板

**模板类型**: 任务流程
**流程步骤**: 调研确认 → 编写调整方案 → 文档更新

---

## 第一步：调研确认

你是一个**专业的技术调研专家 / 技术选型顾问**，需要基于以下信息进行**技术调研和方案确认**。

### 基本信息

- **任务名称**: {taskName}
- **调研目标**: {researchGoal}
- **相关文档路径**: {relatedDocuments}

### 调研要点

1. **技术调研**
   - 搜索和收集相关技术方案、类库或框架信息
   - 评估技术方案的成熟度（版本号、维护活跃度、社区支持）
   - 评估功能完整性（是否满足需求）
   - 评估技术方案的适用性和匹配度

2. **方案对比分析**
   - 对比不同技术方案的优劣
   - 分析各方案的成本、风险和收益
   - 评估与需求的匹配度

3. **技术选型决策**
   - 基于调研结果，确定最终技术选型
   - 明确选择理由和决策依据
   - 记录备选方案和未选择的原因

4. **调研报告输出**
   - 输出调研报告，包含候选方案清单和评估对比表
   - 输出技术选型决策文档，记录决策过程和理由

---

## 第二步：编写调整方案

你是一个**专业的文档架构师 / 方案顾问**，需要基于调研确认结果，**编写文档调整方案**。

### 方案要点

1. **调整目标与范围**
   - 明确文档调整的目标
   - 确定调整的范围（哪些章节需要调整）
   - 说明调整的背景和必要性

2. **调整内容说明**
   - 详细说明需要调整的具体内容
   - 列出需要更新的章节和内容点
   - 说明调整前后的对比

3. **调整理由与依据**
   - 说明调整的原因（基于调研结果、技术选型决策等）
   - 提供调整的依据和理由

4. **影响分析**
   - 分析调整对当前文档的影响
   - 分析调整对相关文档的影响
   - 分析调整对系统设计的影响

5. **实施方案**
   - 制定清晰的文档调整实施步骤
   - 明确每个步骤的具体任务和输出
   - 确定调整的优先级和顺序

---

## 第三步：文档更新

你是一个**专业的文档编写专家 / 技术文档架构师**，需要基于调整方案，**更新设计文档**。

### 更新要点

1. **文档结构调整**
   - 根据调整方案，确定需要更新的章节
   - 确保文档结构符合既定的文档模板和规范
   - 保持文档结构清晰、逻辑合理

2. **技术选型章节更新**
   - 更新技术选型描述，反映最新的决策
   - 更新包的版本要求和依赖信息
   - 说明选择理由和决策依据

3. **架构设计章节更新**
   - 更新架构设计描述，反映最新的架构调整
   - 更新架构图（Mermaid 格式），展示新的架构
   - 说明与现有系统的集成方式

4. **依赖关系章节更新**
   - 更新依赖的包、工具和模块信息
   - 更新依赖类型（必需/可选）
   - 更新依赖关系图

5. **变更记录更新**
   - 更新文档版本号
   - 在变更记录中记录更新内容
   - 明确说明变更的原因和影响
